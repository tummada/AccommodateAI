## Context

Security audit `security-check-output/20260420_091511/security_report_human.md` flagged HIGH-2:
- `infra/backup.sh` creates `.sql.gz` (plain compressed SQL) → uploads to Cloudflare R2
- If R2 bucket leaks or R2 API key is stolen → entire customer leads DB content is readable

## Goal

Encrypt the backup archive with GPG (asymmetric — public key encryption) before uploading, so that:
- VPS holds only the PUBLIC key → can encrypt but cannot decrypt (a VPS compromise does not grant restore ability)
- Owner holds PRIVATE key offline → can decrypt for restore
- Even if R2 contents leak, attacker cannot read them without private key

## Design (recommended — agent may propose alternative with justification)

1. Generate GPG keypair ONCE on Lead workstation (NOT on VPS):
   ```
   gpg --batch --passphrase "$OWNER_PASSPHRASE" --quick-gen-key "VOLLOS Backup <backup@vollos.ai>" rsa4096 encrypt 0
   gpg --armor --export backup@vollos.ai > vollos-backup-public.asc
   gpg --armor --export-secret-keys backup@vollos.ai > vollos-backup-private.asc   # → move to offline storage
   ```
   NOTE: DevOps should NOT run this step. Instead produce a `RUNBOOK-key-setup.md` inside `_workspace/T-040/` explaining the steps for owner to run manually. DevOps only wires up the consumption side.

2. VPS side (code-only change — DevOps implements):
   - Add `infra/backup-public.asc` (committed — PUBLIC key is safe to commit)
   - Update `backup.sh`: pipe `pg_dump | gzip | gpg --encrypt --recipient backup@vollos.ai --trust-model always` → write `.sql.gz.gpg`
   - Keep existing retention + R2 upload logic; just change the filename extension
   - Add `set -euo pipefail` if missing; ensure exit on pipeline failure (`set -o pipefail` or use `PIPESTATUS`)

3. Restore script:
   - Create/update `infra/restore.sh` that pulls `.sql.gz.gpg` from R2, runs `gpg --decrypt | gunzip | psql`
   - Clearly label this as "run on admin workstation only, NOT on VPS"

4. `.env.example` + `README.md`:
   - Document new setup steps
   - Verify no secrets in .env.example (just key names)
